1
Tools everyone needs
▶
Install Obsidian
Download from obsidian.md and install it. This is where all our documentation, lore, design docs, and meeting notes live. Everyone uses it regardless of role. Watch: Get started with Obsidian
Create a GitHub account
Go to github.com and sign up for a free account. Share your username with the code lead so they can invite you to the project.
Install GitHub Desktop (or git CLI if you prefer)
Download from desktop.github.com. Install and sign in with your GitHub account.
Prefer the command line? If you already use git CLI, that works too. GitHub Desktop is just the beginner-friendly option.
Install Git LFS
Download from git-lfs.com and install it. Open a terminal (search "Terminal" on Mac or "Command Prompt" on Windows) and type: git lfs install
Never used a terminal? Just open it, type those exact words, press Enter. You should see "Git LFS initialized". Close it. That's the only time you'll need it.
Join the Discord server
Download Discord if you don't have it. Ask the team lead for the notaBard server invite link.
2
Your role-specific tools
▶
Get comfortable with Obsidian Story
Obsidian is your main tool. Start a note, write some lore, play around with it. Everything you create will eventually live here. Templates will be made once everything is ready. Watch: Obsidian for beginners
Install Godot 4.5.2 Story
Download Godot 4.5.2 stable (standard version, not .NET) from godotengine.org/download/archive. You'll use this to work with Dialogic 2 — the visual dialogue editor where your writing becomes playable conversations.
Install Dialogic 2 plugin Story
In Godot, go to the AssetLib tab at the top. Search for Dialogic, download and install it. This is how you'll build the King's commission scenes, Elder dialogues, Carpenter conversations, and Ruse interactions — all in a visual editor, no code needed. Read: Dialogic 2 documentation
Install Godot 4.5.2 Mechanics
Download Godot 4.5.2 stable (standard version, not .NET) from godotengine.org/download/archive. Have a poke around and get familiar with the editor. Watch: Getting started with Godot
Install MetSys and LimboAI plugins Mechanics
In Godot, go to the AssetLib tab at the top. Search for Metroidvania System and LimboAI. Download and install both from the plugin tab.
Use Obsidian for design ideas Mechanics
Also worth having Obsidian set up to document your design ideas before you build them. Write down ability concepts, enemy patterns, rhythm system ideas.
Install FMOD Studio Music
Go to fmod.com, create a free account, and download FMOD Studio. It's free for indie projects. We'll be using it for all our in-game audio. Watch: Getting started with FMOD
Start exploring and jotting down ideas Music
If you've got ideas for the feel and mood of the soundtrack, jot them down somewhere. We'll get them into Obsidian soon. Think about the 80 BPM master tempo, regional feels, and how music should react to player alignment.
Get familiar with Aseprite (or LibreSprite) Art
Aseprite is our primary pixel art tool. It's not free but not expensive — $19.99 from aseprite.org or on Steam.

If you want to try a free version first, look at LibreSprite. It looks and feels the same, and you're only missing some of the more advanced features. Watch: Getting started with Aseprite
Start thinking about art direction Art
If you want to get ahead, start thinking about the art direction and feel free to do some concept sketches. Once Obsidian is set up you'll have a space to document the art style too. References: Planet of Lana, Hollow Knight, Summerhouse, Eastward.
Install Godot 4.5.2 Coding
Download Godot 4.5.2 stable (standard version, not .NET) from godotengine.org/download/archive.
Get to know GDScript Coding
GDScript is the programming language we use in Godot. It's similar to Python and very beginner-friendly. Start: Learn GDScript interactive app
Make sure GitHub Desktop (or git CLI) is ready Coding
Once the repo is up you'll get an invite and can clone it straight away. Make sure GitHub Desktop is installed and signed in from Section 1.
3
Get the project
▶
Accept the GitHub invitation
Check your email for an invitation to the notabard repository. Click the link to accept. If you haven't received it, share your GitHub username with the code lead.
Clone the project
Open GitHub Desktop, click File then Clone Repository, find notabard in the list, choose where to save it, and click Clone.
Where to save it? Somewhere easy to find, like Documents. Avoid cloud-synced folders (iCloud, OneDrive, Google Drive) — they cause conflicts with Git.
Open the project in Godot
Open Godot 4.5.2, click Import, browse to the cloned folder, select project.godot, and click Import and Edit. Press F5 to run the test scene and confirm everything works.
Open the docs vault in Obsidian
Open Obsidian, click Open folder as vault, navigate to the cloned notabard folder, and select the docs folder. You should see the Dashboard with design pillars, team info, and milestone tasks.
4
Learn the daily workflow
▶
Practice: Pull the latest changes
Open GitHub Desktop and click Fetch origin. If there are changes, click Pull origin. Do this every time before you start working.
Practice: Make a change and push it
In Obsidian, open the Dashboard and add your name to the Team table. Save. Go to GitHub Desktop — you'll see the change. Write "Added my name to the team list", click Commit to main, then Push origin.
You just made your first contribution. This pull, work, commit, push cycle is what you'll do every time.
Try the Discord bots
In the Discord server, try these commands:
!tech What version of Godot are we using?
!lore Who is the companion cat?
!bothelp for all commands
5
Your first task
▶
Write some lore in Obsidian Story
Start a note and write something — character voice samples for Eos, a scene description for the Royal City, lore about one of the Five Movements. Anything goes. Get comfortable writing here.
Explore Dialogic 2 in Godot Story
Open Godot, find the Dialogic tab, and try creating a simple test conversation. Add a character, write a few lines, add a branching choice. This is how all the game's dialogue will be built. Read: Dialogic 2 getting started
Explore MetSys and LimboAI Mechanics
Open the sample projects that come with MetSys and LimboAI. Play with them. Write design notes in Obsidian about how they could map to notaBard — room transitions, cat behavior, enemy patterns.
Sketch a mood piece at 80 BPM Music
In FMOD Studio or your DAW, create a short mood piece at 80 BPM. Doesn't need to be final — just capture the feel. Think about how the Royal City should sound: regal, warm, steady quarter-note pulse.
Do a concept sketch or style test Art
Sketch some concepts for the art direction. Try a pixel art test piece — maybe Eos with his cello, or the companion cat. Reference the GDD's visual influences: Planet of Lana, Hollow Knight, Summerhouse, Eastward.
Work through GDScript basics Coding
Work through the GDQuest tutorial. Focus on understanding scenes, nodes, signals, and basic scripting. Once the repo is up you can clone it and start experimenting. Start: Learn GDScript interactive app
